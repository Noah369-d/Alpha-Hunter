## 🚀 快速开始

### 1. 启动服务器
```bash
# 双击运行
start.bat

# 或使用命令行
node server.cjs
```

### 2. 访问应用

- **测试页**: http://localhost:8000/test.html

### 3. 开始使用
1. 输入股票代码（如: AAPL,MSFT,GOOGL）
2. 点击"扫描"按钮
3. 选择股票查看图表和分析
4. 使用绘图工具和交易管理功能

## ✨ 核心功能

### 1. 模块化架构 ✅
```
src/
├── app.js              # 主应用 (完整实现)
├── utils/
│   ├── ta.js           # 技术指标库 (10+指标)
│   ├── api.js          # API工具 (带重试)
│   ├── storage.js      # 存储管理
│   ├── helpers.js      # 辅助函数
│   └── errorHandler.js # 错误处理系统
└── workers/
    └── indicator-worker.js  # 计算引擎
```

### 2. 完整UI界面 ✅
- **顶部工具栏**: 策略选择、周期切换、主题切换、快捷列表
- **左侧面板**: 股票列表（筛选、排序、自选、情绪标记）
- **中间面板**: 图表（K线、指标、绘图工具）
- **底部面板**: 
  - 日志：策略分析和实时解盘
  - 资讯：RSS新闻订阅
  - SOP：操作标准流程
  - 资产：模拟交易和持仓管理

### 3. 图表系统 ✅
- **K线图**: LightweightCharts专业图表
- **指标线**: 
  - Holo策略：双轨系统（快轨/慢轨）
  - Envelope策略：外包线（上轨/中轨/下轨）
  - T3策略：T3均线
  - CRT策略：陷阱识别
- **交易线**: 成本线、止盈线、止损线
- **绘图工具**: 水平线、趋势线、清除功能

### 4. 交易管理 ✅
- **模拟交易**: 成本、数量、止盈、止损输入
- **价格预警**: 触及止盈/止损时闪烁提示
- **持仓管理**: 添加、删除、盈亏计算、占比分析
- **实时更新**: 自动更新持仓价格和盈亏

### 5. 策略系统 ✅
- **Holo (机构猎手)**: 双轨共振，捕捉机构动向
- **CRT (陷阱识别)**: 识别假突破和假跌破
- **Envelope (外包线)**: 均值回归策略
- **T3 (趋势跟踪)**: T3均线+MACD组合

### 6. 数据管理 ✅
- **多市场**: 美股、港股、A股
- **多周期**: 15m、1h、4h、1d、1wk
- **批量扫描**: 支持多股票同时分析
- **自动刷新**: 15分钟倒计时自动更新
- **本地存储**: 自选、情绪、持仓持久化

## 📝 使用指南

### 基础操作

#### 1. 扫描股票
```
输入: AAPL,MSFT,GOOGL
点击: 扫描
结果: 显示股票列表和分析
```

#### 2. 查看图表
```
点击: 股票列表中的任意股票
结果: 显示K线图和指标
```

#### 3. 使用绘图工具
```
悬停: 图表左侧显示工具栏
点击: 水平线/趋势线图标
操作: 在图表上点击绘制
清除: 点击垃圾桶图标
```

#### 4. 设置交易
```
输入: 成本、数量、止盈、止损
结果: 图表显示价格线
预警: 触及止盈/止损时闪烁
```

#### 5. 管理持仓
```
设置: 交易参数
点击: "加入持仓组合"
查看: 切换到"持仓(Assets)"标签
删除: 点击持仓右侧的"×"
```

### 高级功能

#### 1. 策略切换
```
位置: 顶部工具栏 "STRAT" 下拉框
选择: Holo / CRT / Envelope / T3
效果: 图表指标和SOP指令自动切换
```

#### 2. 周期切换
```
位置: 顶部工具栏周期按钮
选择: 15m / 1h / 4h / 1d / 1wk
效果: 重新加载对应周期数据
```

#### 3. 市场筛选
```
位置: 股票列表顶部
选择: ALL / US / HK / CN
效果: 只显示对应市场的股票
```

#### 4. 自选管理
```
添加: 点击股票左侧的"★"
查看: 点击顶部"★ 自选"按钮
导入: 点击"📂"上传txt/csv文件
```

#### 5. 情绪标记
```
点击: 股票左侧的圆点
状态: 无 → 看涨(绿) → 看跌(红) → 震荡(黄) → 无
用途: 个人研判标记
```
